Request that the engine stop updating this object. 

Call this function in an {\ttfamily \doxylink{class_update_object}{Update\+Object}} -\/derived class to let the engine know you would like to stop updating the object. This function is only accessible to classes deriving from {\ttfamily \doxylink{class_update_object}{Update\+Object}} .

\begin{DoxyNote}{Note}
As the name of the function implies, this puts in a request with the engine for your object to stop being updated. The request is processed by the engine on the same frame this function is called, and calls to {\ttfamily \doxylink{class_update_object_a288ca794056a05fafc9ef19e57532366}{Update\+Object\+::\+Update}} will cease {\bfseries{the}} {\bfseries{following}} frame.
\end{DoxyNote}

Does not return anything.
\end{DoxyReturn}

void Update\+Object\+::\+Update (\begin{DoxyParamCaption}\item[{float}]{delta\+Time}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}}



Update an object. 

The function called by the engine to update an object. This function does nothing by default, and can be overridden by an {\ttfamily \doxylink{class_update_object}{Update\+Object}} -\/derived class. Any behavior that should occur when updating an object should go in the overridden function body.


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em delta\+Time} & The amount of time since the previous frame, in seconds.\\
\hline
\end{DoxyParams}
\begin{DoxyNote}{Note}
This engine is not framerate-\/controlled! A frame will render as soon as it can, meaning the time between frames is unknown. Any behavior (such as movement) need to be multiplied by the provided {\ttfamily delta\+Time} argument in order to appear normal!
\end{DoxyNote}
\begin{DoxyReturn}{Returns}
Does not return anything.
\end{DoxyReturn}
