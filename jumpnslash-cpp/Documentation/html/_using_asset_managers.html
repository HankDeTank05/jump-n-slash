Using the Asset Managers

Overview

The asset managers allow you to quickly and easily manage and access any assets you need to use in your game. They all have the same basic functionality:

  • load an asset (makes an asset accessible in the code)
  • get an asset (access/use an asset that's been loaded)

Texture Manager

The Texture Manager handles any 2D image-based assets. It is the simplest asset manager and also the most essential.

Note
Sprites are created using textures, so any sprite you want to create must be first loaded in as a texture.

Loading a Texture

In order to load a texture, you must provide at least two pieces of information to the TextureManager::LoadTexture function:

  • The key (an std::string)
    • This is used to identify the texture when accessing it in the future. It can be whatever you want. (ex: "mario's sprite :)")
  • The file name/location (also an std::string)
    • This is used to locate the texture within the Assets/Textures folder.
    • If your texture is in the Textures folder, and not in any subfolder, simply provide the name of the file (ex: "mario.png");
    • If your texture is in a subfolder within Textures, you need to provide the relative path to the file (ex: "characters/mario.png");
  • (Optional) Whether or not the texture should be smoothed (a bool, true by default)
    • Provide false as the third argument if you don't want any smoothing on your textures. Otherwise, this argument can be omitted.
    • If you're using pixel art that is extremely low-resolution, you probably don't want smoothing.

Getting a Texture

In order to get a texture, you only need to provide one piece of information to the TextureManager::GetTexture function:

  • The key (an std::string)
    • This is the key you specified when loading the texture. The game will crash if you specify a key that does not exist.

Sprite Manager

The Sprite Manager handles, well, sprites! This asset manager is a little more complex, as there are multiple ways to load sprites.

Loading a Sprites

The simplest and easiest way to load a sprite is using two keys:

  • The sprite key (an std::string)
    • Like the texture key, this is the key used to identify the sprite when accessing it in the future. It can also be whatever you want.
  • The texture key (an std::string)
    • This key is used to access a texture which has already been loaded. Like Getting a Texture, the game will crash if you specify a key that does not exist.
  • (Optional) The sub-area of the texture to use for the sprite (an sf::IntRect)
    • Providing an sf::IntRect as the third argument will allow you to use only a portion of the texture when creating the sprite. This is most useful when you want to load one spritesheet as a texture, but create multiple sprites from it. (See the example code from the previous tutorial)
Note
There are four different ways to load a sprite. If you're interested, check out the documentation for SpriteManager::LoadSprite.

Getting a Sprite

Similar to getting a texture, you only need to provide one argument to access a sprite:

  • The key (an std::string)
    • This is the key you specified when loading the sprite. The game will crash if you specify a key that does not exist.
